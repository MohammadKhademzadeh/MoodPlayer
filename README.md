
 🎵 MoodPlayer - پخش‌کننده هوشمند موسیقی برای لینوکس

MoodPlayer یک اپلیکیشن گرافیکی (GUI) سبک برای لینوکس است که با استفاده از فریم‌ورک Qt6 و زبان ++C نوشته شده است. این برنامه با مانیتور کردن پنجره فعال شما (مثلاً VS Code) و میزان مصرف پردازنده (CPU)، به‌طور خودکار تصمیم می‌گیرد که چه سبک موسیقی برای شما پخش شود تا تمرکز شما را افزایش دهد یا با فعالیت سیستم هماهنگ باشد.

✨ ویژگی‌ها
- 🖥️ **تشخیص پنجره فعال:** تشخیص برنامه در حال اجرا (پشتیبانی از X11).
- ⚙️ **مانیتورینگ پردازنده:** محاسبه لحظه‌ای درصد مصرف CPU از طریق خواندن `/proc/stat`.
- 🎧 **پخش خودکار موسیقی:** تغییر آهنگ پخش‌شده بر اساس حالت (تمرکز، فعالیت سنگین، فعالیت سبک).
- 🐧 **سازگار با لینوکس:** تست شده روی Arch Linux و GNOME.

---

 📦 پیش‌نیازها

برای کامپایل و اجرای این برنامه، باید کتابخانه‌های زیر روی سیستم شما نصب باشند.

**در Arch Linux و مشتقات آن:**
```bash
sudo pacman -S base-devel cmake qt6-base qt6-multimedia libx11
```

**در Ubuntu/Debian و مشتقات آن:**
```bash
sudo apt update
sudo apt install build-essential cmake qt6-base-dev qt6-multimedia-dev libx11-dev
```

---

 🛠️ نحوه نصب و کامپایل

پروژه از سیستم ساخت CMake استفاده می‌کند. برای کامپایل کردن کافیست مراحل زیر را در ترمینال دنبال کنید:

۱. مخزن را کلون کنید:
```bash
git clone https://github.com/USERNAME/MoodPlayer.git
cd MoodPlayer
```
*(به جای `USERNAME` نام کاربری گیت‌هاب خود را وارد کنید)*

۲. یک پوشه برای فایل‌های بیلد بسازید و وارد آن شوید:
```bash
mkdir build
cd build
```

۳. پروژه را پیکربندی و کامپایل کنید:
```bash
cmake ..
make
```

۴. برنامه را اجرا کنید:
```bash
./MoodPlayer
```

⚙️ تنظیمات و پیکربندی موسیقی

برای اینکه برنامه آهنگ‌های شما را پخش کند، باید مسیر فایل‌های موسیقی (mp3 و ...) را در کد مشخص کنید. 

فایل `MainWindow.cpp` را باز کنید و به تابع `decideMusic` بروید. در این تابع می‌توانید مسیر فایل‌های خود را جایگزین کنید:

```cpp
// برای حالت تمرکز (مثلاً وقتی VS Code باز است)
QString focusMusic = "file:///home/youruser/Music/focus.mp3"; 

// برای فعالیت سنگین (وقتی CPU بالای 70% است)
QString intenseMusic = "file:///home/youruser/Music/intense.mp3";

// برای فعالیت سبک
QString chillMusic = "file:///home/youruser/Music/chill.mp3";
```
بعد از تغییر مسیرها، برنامه را مجدداً با دستورات `cmake ..` و `make` کامپایل کنید.

 ⚠️ نکات مهم (مخصوص کاربران GNOME و Wayland)

محیط‌های دسکتاپ مدرن مثل گنوم به‌صورت پیش‌فرض از پروتکل **Wayland** استفاده می‌کنند. Wayland به دلایل امنیتی اجازه نمی‌دهد برنامه‌ها به راحتی پنجره فعال سیستم را ببینند.
- برنامه‌های مبتنی بر Electron (مثل VS Code) معمولاً روی XWayland اجرا می‌شوند و MoodPlayer می‌تواند آن‌ها را تشخیص دهد.
- اگر می‌خواهید تمام برنامه‌های سیستم (مثل ترمینال یا فایل‌منیجر) به درستی تشخیص داده شوند، پیشنهاد می‌شود هنگام لاگین در صفحه GDM، گزینه **"GNOME on Xorg"** را انتخاب کنید.
